# PyTorch Transformer Interpretability: Belief-State Geometry in Non-Ergodic Sequence Models

**AI safety / mechanistic interpretability project: how a PyTorch transformer represents hidden latent structure during next-token prediction.**

> **TL;DR** — I trained a small decoder-only transformer, implemented from scratch in PyTorch, on non-ergodic synthetic sequences generated from a mixture of hidden Markov processes. I then probed the residual stream to test whether the model internally represents latent belief states and process identity. Result: the ground-truth belief state is almost perfectly linearly decodable from activations (R² > 0.989), while process identity is only partially decodable (~60% vs. 33% chance) — the model builds a compact, shared geometry that prioritizes local predictive state over clean global source separation.

Based on and extending [**"Transformers Learn Factored Representations"**](https://arxiv.org/abs/2602.02385) (Shai et al.), which shows transformers can factor their world into orthogonal residual-stream subspaces. This project tests what happens when the factor itself — which hidden process generated the sequence — is latent and never labeled.

---

## Why it matters

Real text is never generated by one stationary source — it's a mixture of hidden regimes (topic, speaker, task, style), each internally consistent but collectively non-ergodic. A model has to infer *which* regime it's in while tracking *where* it is within that regime — a two-level inference problem that in-context learning solves implicitly. This project is a small, fully-labeled testbed for studying that problem mechanistically: since the true latent variables are known exactly, I can directly check whether next-token prediction alone induces an interpretable internal geometry, and where it falls short.

---

## Background

Data comes from three Mess3 hidden Markov processes (3 hidden states, 3 symbols each), whose exact Bayesian belief state can be computed in closed form and lives on a 2D probability simplex.

| Process | α    | x    | Character |
|---------|------|------|-----------|
| P1      | 0.95 | 0.02 | Belief concentrated near simplex vertices |
| P2      | 0.90 | 0.05 | Intermediate spread |
| P3      | 0.85 | 0.12 | Broadest, multi-scale belief geometry |

Each training sequence is generated entirely by **one** process, with no label — making the overall dataset non-ergodic. The model must infer both active process and within-process belief state from tokens alone.

---

## Method

- **Model**: decoder-only transformer built from scratch in PyTorch (explicit causal self-attention, no HF library) — 2 blocks, `d_model=64`, 2 heads, MLP width 256, ~101K params, context length 16.
- **Training**: next-token cross-entropy, Adam, lr `5e-4`, batch 256, 8,000 steps.
- **Analysis**: residual-stream activations extracted at every layer/position; PCA + effective dimensionality (95% CEV); linear probes decoding ground-truth belief state and process identity.

---

## Pre-registered prediction

Each process's belief vector lives in a 2D simplex (3 states → 2 degrees of freedom). The full joint over (process, hidden state) is a 9-dim distribution — an 8-simplex. Treating process identity and within-process state as two pseudo-factors gives:

- **Lower bound: (3−1) + (3−1) = 4D** if the network shares one compact belief subspace across processes.
- **Upper bound: 3 × 2 = 6D** if it allocates a fully separate orthogonal subspace per process.

I predicted the answer would land near the 4D lower bound — a **partially shared geometry**: one reused belief subspace, with process identity only weakly, imperfectly encoded. (Alternatives considered: fully factored/orthogonal 4D with clean separability of both factors, or a fully joint 6–8D encoding with no factorization at all.)

---

## Results

1. **Model learned real structure** — loss converged to ≈0.74, well below the uniform baseline `log(3) ≈ 1.099`.
2. **Residual stream compresses** — effective dimensionality drops from ~12D (raw embedding) to ~3D after layer 1, settling at **~4D** after layer 2 — matching the predicted lower bound.
3. **Belief state is almost perfectly linear** — R² > 0.989 across all context positions when probing for the ground-truth Bayesian belief.
4. **Process identity is only partially linear** — ~60% linear decoding accuracy by late context (vs. 33% chance) — well above chance, far from ceiling.
5. **Geometry builds up with context** — early positions are organized only by the last token seen; by late context, activations form a smooth, low-dimensional structure where nearby points correspond to nearby belief states.

---

## Interpretation

The core finding is an asymmetry: **belief state is almost perfectly linearly represented; process identity is not.** The transformer prioritizes the local predictive state it needs for next-token prediction over clean global source disambiguation — confirming the "partially shared" hypothesis over full factorization. In larger LLMs, an analogous effect could explain prompt-sensitivity: a model can track local context well without ever fully committing to a single task/domain/instruction regime. This mirrors a problem I've worked on in computational neuroscience — using GLM-HMMs to infer latent behavioral regimes in animals from observations alone — and part of what makes this setup interesting to me is that it's a minimal, tractable transformer analogue of the same nested-inference problem.

## Possible extensions

- Scale to more hidden processes and vary how similar they are, to test predicted effects on identity decoding.
- Introduce mid-sequence process switches to test whether the residual stream updates dynamically.
- Quantify subspace overlap between per-process representations directly, and test whether sparse autoencoders recover cleaner belief-state features.
